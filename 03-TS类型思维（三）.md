# 一、类型别名

在多个地方，使用同一类型时，可以给对象类型起一个别名：

```typescript
// 类型别名: type
type MyNumber = number
const age: MyNumber = 18

// 给 ID 的类型起一个别名
type IDType = number | string
function printID(id: IDType) {
  console.log(id)
}

// 打印坐标
type PointType = { x: number, y: number, z?: number }
function printCoordinate(point: PointType) {
  console.log(point.x, point.y, point.z)
}
```

# 二、接口

`type` 可用来定义一个对象类型：接口 `interface` 可用来声明对象类型：

```typescript
type PointType = { // 赋值，定义
  x: number
  y: number
  z?: number
}

interface PointType2 { // 声明
  x: number
  y: number
  z?: number
}

function printCoordinate(point: PointType2) {
}
```

# 三、interface 和 type 的区别

`type` 表示类型别名，应用范围广，可用于声明所有类型。

`interface` 表示接口，只能用来声明对象类型。它的特性更多。扩展性更好。

- `interface` 可重复对某个接口来声明属性和方法。`type` 定义的是别名，别名是不能重复的。
- `interface` 之间可以继承/多继承，
- `interface` 可以被类实现/多实现，
- `interface` 可用于泛型类型约束(`<T extends interface>`)

在开发中如何选择：

- 声明对象类型，使用 `interface`，扩展性更强。
- 声明非对象类型，使用 `type`，使用范围更广。

# 四、TS 中的交叉类型

交叉类型，表示多个类型同时满足的类型，使用 `&` 符号；

同时满足两个普通类型，是没有意义的，相当于 `never` 类型。

实际开发中，交叉类型，一般是两个或多个对象类型的交叉。

```typescript
type NewType = number & string // 没有意义，相当于 never

interface IKun {
  name: string
  age: number
}

interface ICoder {
  name: string
  coding: () => void
}

type InfoType = IKun & ICoder

const info: InfoType = {
  name: "zzt",
  age: 18,
  coding: function() {
    console.log("coding")
  }
}
```

# 六、类型断言 as

TypeScript 无法推导具体的类型信息时，需要使用类型断言（Type Assertions）。

- 比如 `document.getElementById` 函数，TypeScript 只知道该函数会返回 `HTMLElement` ，但并不知道它具体的类型：

```typescript
const imgEl = document.getElementById("image") as HTMLImageElement
imgEl.src = "xxx"
imgEl.alt = "yyy"
```

TypeScript 只允许类型断言转换为**更具体**或者**不太具体**的类型版本，此规则可防止直接的不可能的强制转换：

```typescript
const age: number = 18
const age2 = age as string // 会报错。

// 以下写法，TS 类型检测来说是正确的, 但是不推荐这么写
const age3 = age as any
const age4 = age3 as string
console.log(age4.split(" "))
```

类型断言，最主要的目的是，方便对变量（标识符）的使用（赋值，访问属性，调用方法）


# 七、非空类型断言 !.

访问可选类型中的可选属性时，可用可选链（`?.`）访问。

赋值可选类型中的可选属性时，不能用可续连（`?.`），如果该值确定存在，使用非空类型断言（`!.`）赋值。跳过 TS 在编译阶段对它的检测。

```typescript
interface IPerson {
  name: string
  age: number
  friend?: {
    name: string
  }
}

const info: IPerson = {
  name: "zzt",
  age: 18
}

// 访问属性: 使用可选链: ?.
console.log(info.friend?.name)

// 属性赋值:
// 方案一: 类型缩小
if (info.friend) {
  info.friend.name = "kobe"
}

// 方案二: 非空类型断言(谨慎操作, 只有确保 friend 一定有值的情况, 才能使用)
info.friend!.name = "james"
```


# 八、TS 中的字面量类型

单独使用，没有太大意义，一般将多个字面量类型联合起来。

```typescript
type Direction = "left" | "right" | "up" | "down"
const d1: Direction = "left"

type MethodType = "get" | "post"
function request(url: string, method: MethodType) {
}

request("http://codercba.com/api/aaa", "post")
```

## 1.字面量推理

```typescript
type MethodType = "get" | "post"
function request(url: string, method: MethodType) {  
}

request("http://codercba.com/api/aaa", "post") // 正确

const info = {
  url: "xxxx",
  method: "post"
}

request(info.url, info.method) // 错误: info.method 获取的是 string 类型，而非字面量类型

// 解决方案一: info.method 进行类型断言
request(info.url, info.method as "post")

// 解决方案二: 为 info 声明类型注解，指定 info.method 为字面量类型
const info2: { url: string, method: "post" } = {
  url: "xxxx",
  method: "post"
}
request(info2.url, info2.method)

// 解决方案三：使用字面量推导，将 string 类型，推到为 'post' 类型
const info2 = {
  url: "xxxx",
  method: "post"
} as const;
request(info2.url, info2.method)
```

# 九、类型缩小

**类型缩小（Type Narrowing）**，也称**类型收窄**； 

主要是通过改变 TypeScript 的执行路径，缩小模糊的类型到更具体的类型（这个过程称之为**缩小 Narrowing**）;

而用于改变 TypeScript 的执行路径的行为，称之为**类型保护（type guards）**；

常见的类型保护有以下方式：

## 1.typeof 操作符

使用最多的类型保护。

```typescript
function printID(id: number | string) {
  if (typeof id === "string") {
    console.log(id.length, id.split(" "))
  } else {
    console.log(id)
  }
}
```

## 2.平等缩小

一般使用 `switch` 或者像 `===`, `!==`, `==`, `and` `!=` 这样的运算符。

```typescript
type Direction = "left" | "right" | "up" | "down"

function switchDirection(direction: Direction) {
  if (direction === "left") {
    console.log("左:", "角色向左移动")
  } else if (direction === "right") {
    console.log("右:", "角色向右移动")
  } else if (direction === "up") {
    console.log("上:", "角色向上移动")
  } else if (direction === "down") {
    console.log("下:", "角色向下移动")
  }
}
```

## 3.instanceof 操作符

检查一个对象是否是一个类（构造函数）的实例。

```typescript
function printDate(date: string | Date) {
  if (date instanceof Date) {
    console.log(date.getTime())
  } else {
    console.log(date)
  }
}
```

## 4.in 操作符

用于确定对象或者对象的原型链中，是否具有该名称的属性。

```typescript
interface ISwim {
  swim: () => void
}

interface IRun {
  run: () => void
}

function move(animal: ISwim | IRun) {
  // JS 中的写法，TS 中是错误写法，因为 IRun 中没有 swim，
  if (animal.swim) {
    animal.swim()
  } else {
    animal.run()
  }
  
  // 正确写法
  if ("swim" in animal) {
    animal.swim()
  } else if ("run" in animal) {
    animal.run()
  }
}
```




# 十、TS 中的函数类型详解

在 JS 开发中，函数作为一等公民意味着可以：

- 作为别的函数的参数。
- 作为别的函数的返回值。
- 赋值给变量，在变量中传递。
- 存储在数据结构中；

TS 中函数也有自己的类型。表示函数类型，有两种方式。

## 1.函数类型表达式

**函数类型的表达式（Function Type Expressions）**，用于表示函数类型；格式是 `(参数列表) => 返回值`

函数类型的表达式中，函数的形参必写，类型可选（默认为 `any`）。

> 函数中的参数顺序，必传参数，有默认值的参数，可选类型的参数。

```typescript
type BarType = (num1: number, num2?: number) => number

const bar: BarType = (arg: number): number => {
  return 123
}
```

### 1.传入函数的参数校验

TS 对于作为参数传入的函数，它的参数个数少于函数类型中的参数个数时，是不做校验的，两个例子：

案例一：

```typescript
type CalcType = (num1: number, num2: number) => number

function calc(calcFn: CalcType) {
  const num1 = 10
  const num2 = 20
  const res = calcFn(num1, num2) // 此处必须传入函数类型表达式中，规定的 2 个参数
  console.log(res)
}

calc(function (num1: number, num2: number) {
  return num1 + num2
}) // 正常写法

calc(function (num1: number) {
  return num1
}) // 此处传入的匿名函数，并不满足函数类型表达式中规定的 2 个参数，但 TS 不做校验。
```

案例二：

```typescript
const names = ["abc", "cba", "nba"]

// forEach 中的回调函数，函数类型表达式中有 3 个参数，但实际上三个参数并非都是必传的
names.forEach(function(item) {
  console.log(item.length)
})
```

> TS 中语法校验报错与否，取决于它的语法规则（说你行，你就行，不行也行。说不行，就不行，行也不行）。
>
> 从语言设计的角度来理解 TS 的报错机制。TS 的目的，是为了增加类型约束的同时，方便使用（好用和类型检测之间找到一个平衡）。

### 2.严格字面量赋值检测（补充）

```typescript
interface IPerson {
  name: string
  age: number
}

// TS 类型检测报错，对“新鲜”定义的对象，会进行严格的检测
const info: IPerson = {
  name: "zzt",
  age: 18,
  height: 1.88,
  address: "广州市"
}

const p = {
  name: "zzt",
  age: 18,
  height: 1.88,
  address: "广州市"
}
// TS 类型检测通过，对“不新鲜”的对象，检测时会进行”类型擦除“
const info: IPerson = p
```

## 2.函数调用签名

从对象的角度看待函数，函数还可以添加自己的属性。

函数类型表达式并不能支持声明函数属性；此时需要在一个对象类型中，写一个**调用签名（call signature）**；

```typescript
interface IBar {
  name: string
  age: number
  (num1: number): number // 函数调用签名
}

const bar: IBar = (num1: number): number => {
  return 123
}

bar.name = "aaa"
bar.age = 18
bar(123)
```

## 3.如何选择

如果只是描述函数类型本身, 使用**函数类型表达式(Function Type Expressions)**。

如果在描述函数作为对象可以被调用, 同时也有其他属性时, 使用**函数调用签名(Call Signatures)**。
